# SumoSharp parity — SYNTHETIC divergence bundle

**Scenario:** a `netgenerate`-generated **synthetic** 8×8 grid of unsignalized
**priority** junctions, 2 lanes/edge, 120 m edges, dead-end fringe stubs. **No real
road-network geometry** — fully generated by `build.py` (seed 42), shareable. 1360
vehicles: 1200 through (fringe→fringe), 120 park-and-stay (`duration=100000` sink on a
multi-occupant lane-0 parkingArea), 40 depart-from-parked (`departPos="stop"`).

Command (both engines, identical flags, only the binary differs):
```
<bin> -c scenario.sumocfg \
   --fcd-output {van|ss}.fcd.xml --summary-output {van|ss}.sum.xml \
   --tripinfo-output {van|ss}.ti.xml --statistic-output {van|ss}.stat.xml \
   --end 1000 --no-step-log true
```
`sumo` = vanilla SUMO 1.20.0; SumoSharp = `artifacts/sumosharp/linux-x64/sumosharp`.

## No-cheating audit: PASS (both engines)

`audit_nocheat.py grid.net.xml scenario.rou.xml scenario.add.xml ss.ti.xml ss.fcd.xml`
→ 0 birth / 0 death / 0 FCD violations. Every birth/death is at a fringe stub or
off-road in a parkingArea. The divergences below are real engine behaviour.

## Issue 1 — park-and-stay residency  (REPRODUCED)

| metric | vanilla | sumosharp |
|---|---:|---:|
| completed trips (tripinfo count) | 1240 | 1334 |
| park-and-stay completed (of 120) | 0 | 111 |
| peak `running` | 960 | 485 |
| end-of-sim running / stopped | 120 / 120 | 10 / 4 |

The parkingArea sits on **lane 0** of a 2-lane edge. Vanilla lane-changes onto lane 0,
parks the `duration=100000` cars, and keeps them **resident** for the whole run (never
"arrive", never in tripinfo). SumoSharp does not change onto the parking lane, never
brakes for the lane-0 stop (`Engine.cs StopLineConstraint` brakes only when
`stop.LaneId == v.LaneId`), drives off the end of the final parking edge, and is removed
as "arrived" → 111 of 120 complete and leave `running`. Matches the real-net finding
(vanilla resident, SumoSharp removed at arrival).

## Issue 2 — excess deadlock / jam-teleport  (REPRODUCED)

| metric | vanilla | sumosharp |
|---|---:|---:|
| jam-teleports (`teleports jam=`) | 0 | 21 |
| total teleports | 4 | 21 |
| mean relative speed (avg over active steps) | 0.493 | 0.411 |

Vanilla flows through the priority junctions with no jam-teleports; SumoSharp deadlocks
and jam-teleports 21× with a lower mean relative speed — a genuine right-of-way /
gap-acceptance / junction-blocking divergence. Real-net reference: vanilla 1–2,
SumoSharp 33.
